Roll20 로그 커스터마이징 도구

Roll20 로그를 HTML 편집모드를 사용해 백업하기 위한 커스터마이징 페이지입니다.
Blogger, 티스토리 등 HTML 편집 가능한 포스트를 작성할 수 있는 블로그 서비스나 기타 HTML 편집 에디터, 개인홈에서 사용할 수 있습니다.

이 페이지는 Windows10/11 chrome/Edge 환경에서 제작되었습니다.

백업 중 로그가 깨지거나 정상적으로 동작하지 않는 기능이 있다면
↖메뉴를 눌러 문의해 주세요. 문의 전, 가이드 페이지를 확인해 주세요!

커스텀 시트 적용

커스텀 시트를 적용하고 싶을 경우, .css 파일을 업로드 해 주세요.

혹은 'CSS 입력' 버튼을 클릭해 css를 직접 입력하세요.

세부 설정

항목에 커서를 올리면 간단한 설명이 나옵니다.

    • 자신이 작성한 메시지의 색이 다른 플레이어가 작성한 메시지의 색과 같아집니다.
    • 메시지 작성 시각이 표시됩니다.
  • info
    • 일반 명령어 굴림(/r 2d6 등)에서 숫자 뒤에 주사위 아이콘을 표시합니다.
    주의:: 티스토리에 적용할 경우, 노션 설명서를 꼭 읽은 후 적용해 주세요.
    • Roll20화면에서 보여지는 폰트를 사용합니다.
    이 옵션을 해제하면, 사용자의 티스토리에 설정된 기본 글꼴로 출력됩니다.
    • 이 옵션을 선택하면 캐릭터별 색상 설정의 글씨 색상이 그 캐릭터의 전체 색상에 적용됩니다.
    • 이 옵션을 선택하면 캐릭터별 색상 설정의 글씨 색상이 개별 이름에만 적용됩니다. (기본값)

    주의 : 테마 프리셋을 적용하거나 기본 테마로 변경하려면 HTML 입력부터 다시 커스텀해야 합니다.

    적용 버튼을 누른 후 잠시 기다려주세요.

    미리보기

    전체 로그 보기

    체크하면 전체 로그가 아래에 나타납니다.
    메시지 왼쪽의 마이너스(-) 버튼을 누르면 해당 메시지를 삭제할 수 있습니다.

    내용 수정

    '전체 로그 보기' 옵션이 활성화되어 있을 경우에만 작동합니다.
    체크하면 내용을 수정할 수 있습니다. 오탈자 수정 용도로만 사용하기를 권장합니다.
    엔터를 추가하거나 삭제하면 레이아웃이 깨질 수 있습니다. 체크를 해제하면 원래대로 돌아갑니다.

    자동저장 '전체 로그 보기' 옵션이 활성화되어 있을 경우에만 작동합니다.
    자동저장이 활성화 중일 때, 1분 간격으로 미리보기 창의 편집 중인 로그를 브라우저에 자동으로 저장합니다. 브라우저를 변경하거나 캐시를 삭제하면 자동저장된 데이터가 삭제됩니다.
    슬롯이 파란색일 때, 브라우저에 저장되어 있는 로그를 불러오거나 삭제할 수 있습니다.
    자동저장된 로그를 불러오면 현재 편집 중이던 로그를 덮어씁니다.

    플레이어 아바타 이미지가 로딩되지 않을 경우, [ HTML입력-다음 ] 을 한번 더 누르고 잠시 기다려 주세요.

    업로드한 커스텀 시트와 CSS 코드, 세부 설정, 캐릭터별 색상 설정 그리고 변경한 프로필 사진이 백업됩니다.
